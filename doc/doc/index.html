Introduction

General Architecture for Text Engineering or GATE is a Java suite of tools originally developed at the University of Sheffield beginning in 1995 and now used worldwide by a wide community of scientists, companies, teachers and students for all sorts of natural language processing tasks, including information extraction in many languages.

Now combine it with Textalytics and create the most impressive tools!

General Parameters

Name Description
inputASName Name of the AnnotationSet which will be used as input. If both this field and inputASTypes are unset, the whole document will be processed.
inputASTypes

This parameter is defined as a way to filter which annotations you want to process.

It allows expressions that follow the pattern AnnotationType.FeatureName==FeatureValue. Not every element in the pattern has to appear. For example, you may want to specify just Type or Type.FeatureName. Only those annotations that match the pattern will be used as input for the application.

Hint: Take into account that this option will take longer to run because it will process individually the annotations matching the input.
debug In case of system failure, set it to true to get things logged and send us the log's output.
outputASName

Name given to the group of Annotations that this Processing Resource will produce.

New Annotations will be created in the AnnotationSet defined by outputASname.

The Annotations created follow the structure of the API specification. Sometimes you may find nested objects in the specification. Those objects are flattened using ";" as 1st level separator and "|" as 2nd level separator.

Textalytics API

Language Identification API (v1.1)

Language Identification is Textalytics' solution to automatic language detection for texts obtained from any kind of source. Through statistic techniques based on N-grams evaluation, more than 60 languages are correctly identified, helping to automatize multilingual sourced applications either by correcting a manual detection preprocessing or by avoiding it altogether.

Output

The parameter outputASname is not defined for this resource. Thus, the output of the system will depend on whether you defined the input parameters or not:

  • If you selected inputASName or inputASTypes, the output will be included as new Features in the Annotations that match your query.
  • If you left inputASName and inputASTypes unset, the output will be included as new Document Features inside each processed document.

API Documentation

If you need more information about the parameters, please refer to the Language Identification API v1.1.

Text Classification API (v1.1)

Text Classification is Textalytics' solution for automatic text classification according to pre-established categories defined in a model. The algorithm used combines statistic classification with rule-based filtering, which allows to obtain a high degree of precision for very different environments.

Output

The parameter outputASname is not defined for this resource. Thus, the output of the system will depend on wheter you defined the input parameters or not:

  • If you selected inputASName or inputASTypes, the output will be included as new Features in the Annotations that match your query.
  • If you left inputASName and inputASTypes unset, the output will be included as new Document Features inside each processed document.

API Documentation

If you need more information about the parameters, please refer to the Text Classification API v1.1.

Topics Extraction API (v1.2)

Topics Extraction is Textalytics' solution for extracting the different elements present in sources of information. This detection process is carried out by combining a number of complex natural language processing techniques that allow to obtain morphological, syntactic and semantic analyses of a text and use them to identify different types of significant elements.

API Specification

If you need more information about the parameters, please refer to the Topics Extraction API v1.2.

Limitations

We do not provide yet Annotations for the relation_list tag in the current version of the plugin.

Lemmatization, POS and Parsing API (v1.2)

Lemmatization, PoS and Parsing is the name of Textalytics' API for the different basic linguistic modules.

Even though it is simple in name, the parser contains a myriad of functionalities derived from the complete morphosyntactic and semantic analysis it carries out. Instead of including different APIs to obtain all the possible features provided by this analysis, features are grouped in different operating modes, allowing the user to take advantage of as many of them as he wishes and to combine them with other Textalytics' features, such as Topics Extraction.

API Specification

If you need more information about the parameters, please refer to the Lemmatization, POS and Parsing API v1.2.

Spell, Grammar and Style Proofreading API (v.1.1)

Spelling, Grammar and Style Proofreading is our solution for automatic proofreading. This module uses multilingual Natural Language Processing technology to check the spelling, grammar and style of your texts with high accuracy, in order to provide precise and up-to-date suggestions and educational explanations based on references. It also provides a number of configuration parameters that allow the user to obtain the corrections in different formats and to customize the behavior for certain types of errors.

API Specification

If you need more information about the parameters, please refer to the Spell, Grammar and Style Proofreading API v.1.1.

Sentiment Analysis API (v1.1)

Sentiment Analysis is Textalytics' solution for performing a detailed multilingual sentiment analysis of texts from different sources.

The text provided is analyzed to determine if it expresses a positive/negative/neutral sentiment; to do this, the local polarity of the different sentences in the text is identified and the relationship between them evaluated, resulting in a global polarity value for the whole text.

API Specification

If you need more information about the parameters, please refer to the Sentiment Analysis API v1.1.

Limitations

We do not provide yet Annotations for the entity_list and concept_list tags in the current version of the plugin.

Install

To Install the Textalytics plugin in GATE:

  1. Open the GATE Developer.
  2. Click on File > Manage CREOLE Plugins... to open the CREOLE Plugin Manager.
  3. Go to the Configuration tab:
    • Fill the User Plugin Directory: with a local directory where you have write permissions.
    • Click on the + symbol from Plugin Repositories: and add the Textalytics repository url: http://textalytics.com/files/download/GATE/gate-update-site.xml
    • Click Apply All.
  4. Go to the Available to install tab, that must be active now:
    • Look for the Textalytics Plugin and mark it to Install. Then click Apply All.
  5. Go to the Installed Plugins tab:
    • Look for the Textalytics Plugin and mark it to Load Now or/and Load Always.
    • Click on Apply All and then Close.
  6. Once loaded, you can add Textalytics APIs as Processing Resources in the left panel or load the Textalytics.xgapp from your User Plugin Directory by going to Applications > Restore Application from file...
